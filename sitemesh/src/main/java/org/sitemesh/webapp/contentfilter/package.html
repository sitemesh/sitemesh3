Provides a mechanism for intercepting content generated by web-apps and making modifications to it.

There are 2 mechanisms for doing this:

  1. {@link HttpServletResponseBuffer} wraps a {@link jakarta.servlet.http.HttpServletResponse} and to forwards all content written to the response to an intermediate buffer which can be accessed. This can be passed to anything that accepts a standard HttpServlerResponse (such as a {@link jakarta.servlet.RequestDispatcher}).
  2. {@link ContentBufferingFilter} is a Servlet {@link jakarta.servlet.Filter} that automatically applies the {@link HttpServletResponseBuffer} to requests. To use it, subclass it and implement application specific functionality.

Both of these are suitable for working with text content, but not binary content.
